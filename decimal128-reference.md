<h2><a id="user-content-introductory-example" class="anchor" aria-hidden="true" href="#introductory-example"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Introductory example</h2>
<p><code>Decimal128</code> is a new numerical type proposed for JavaScript which can be used to represent decimal quantities, like money.</p>
<p>For example, to add up a bill with a number of items, and add sales tax:</p>

<p>Here, you can see several elements of <code>Decimal128</code> at work:</p>
<ul>
<li>Create a <code>Decimal128</code> as a literal, e.g., <code>1.25m</code>, or convert one from another type, as <code>Decimal128(value)</code>.</li>
<li>Add and multiply <code>Decimal128</code>s with the <code>+</code> and <code>*</code> operators.</li>
<li>Round decimals with <code>Decimal128.round</code>, based on an options bag describing how to round.</li>
</ul>
<p>This article describes how these work in more detail, and the rest of the <code>Decimal128</code> API.</p>
<h2><a id="user-content-what-does-decimal128-represent" class="anchor" aria-hidden="true" href="#what-does-decimal128-represent"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>What does Decimal128 represent?</h2>
<p><code>Decimal128</code> represents a base-10 decimal number, internally represented as an IEEE 754 128-bit decimal. Its precision is defined by 34 decimal digits of significand and an exponent range from -6143 to +6144. The value of this type is calculated as follow <code>s * (&lt;significand&gt; * 10 ** &lt;exponent&gt;)</code>, where <code>s</code> rerpesents the sign of the number and can be either <code>1</code> or <code>-1</code>.</p>
<p>IEEE 754 128-bit decimal allows represent different precisions of the same value considering trailing zeros (i.e it is possible to represent both <code>2.10000</code> and <code>2.1</code>), however Decimal128 values do not represent precision apart form its value. It means that two Decimal128 values are equal when they represent the same mathematical value (e.g. <code>2.10000m</code> is the same number as <code>2.1m</code> with the very same precision).</p>
<h2><a id="user-content-creating-decimal128-values" class="anchor" aria-hidden="true" href="#creating-decimal128-values"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Creating Decimal128 values</h2>
<p>There are 2 main ways to create new Decimal128 values. We can use <strong>literals</strong> or <strong>constructors</strong>.</p>

<p>With the exception of addition operator <code>+</code>, mixing Decimal128 and other primitive types results in a <code>TypeError</code> (see <a href="https://github.com/tc39/proposal-decimal/issues/39">issue #39</a> for reasoning behind this design decision).</p>

</code></pre>
<h4><a id="user-content-rounding-on-arithmetic-operations" class="anchor" aria-hidden="true" href="#rounding-on-arithmetic-operations"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Rounding on arithmetic operations</h4>
<p>It is important to notice that Decimal128 precision is limited to 34 digits and every arithmetic operations like addition or multiplication can cause rounding torwards that precision. The rouding algorithm used on Decimal128 arithmetics is the <code>half even</code> where it rounds towards the "nearest neighbor". If both neighbors are equidistant, it rounds towards the even neighbor. It is listed bellow examples of rounding for each operation:</p>
<pre><code>let sumRounded = 1e35m + 1m;
print(sumRounded); // prints 1e35

let subRounded = -1e35m - 1m;
print(subRounded); // prints -1e35

let mulRounded = 1m * 1e-400m;
print(mulRounded); // prints 0

let mulRounded = 1m / 1e400m;
print(mulRounded); // prints 0
</code></pre>
<p>Operations that can cause roundings are the ones where the precision of result is greater than 34.</p>

<ul>
<li><code>value</code>: A Decimal128 value. If the value is from another type, it throws <code>TypeError</code>.</li>
<li><code>options</code>: It is an object indicating how the round operation should be performed. It is an object that can contain <code>roundingMode</code> and <code>maximumFractionDigits</code> properties.
<ul>
<li><code>maximumFractionDigits</code>: This options indicates the maximum of fractional digits the rounding operation should preserve.</li>
<li><code>roundingMode</code>: This option indicates which algorithm is used to round a given Decimal128. Each possible option is described below.
<ul>
<li><code>down</code>: round towards zero.</li>
<li><code>half down</code>: round towards "nearest neighbor". If both neighbors are equidistant, it rounds down.</li>
<li><code>half up</code>: round towards "nearest neighbor".  If both neighbors are equidistant, it rounds up.</li>
<li><code>half even</code>: round towards the "nearest neighbor". If both neighbors are equidistant, it rounds towards the even neighbor.</li>
<li><code>up</code>: round away from zero.</li>
</ul>

<h3><a id="user-content-decimal128reminderlhs-rhs--options" class="anchor" aria-hidden="true" href="#decimal128reminderlhs-rhs--options"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Decimal128.reminder(lhs, rhs [, options])</h3>
<p>This function returns the reminder of dividing <code>lhs</code> by <code>rhs</code>, applying the rounding based on <code>options</code> object, if given. <code>options</code> is an options bag that configures a custom rounding for this operation.</p>
<ul>
<li><code>lhs</code>: A <code>Decimal128</code> value. If the value is from another type, it throws <code>TypeError</code>.</li>
<li><code>rhs</code>: A <code>Decimal128</code> value. If the value is from another type, it throws <code>TypeError</code>.</li>
<li><code>options</code>: It is an object indicating how the round operation should be performed. It's the same options bag object described on <a href="#decimal128roundvalue--options">Decimal128.round</a>. If it's not given, the operation will use the same rounding rules of <code>%</code> binary operator described on <a href="#rounding-on-arithmetic-operations">Rounding on arithmetic operations</a> section.</li>
</ul>

<p>There is a <code>Decimal128.prototype</code> that includes utility methods.</p>
<h3><a id="user-content-decimal128prototypetostring" class="anchor" aria-hidden="true" href="#decimal128prototypetostring"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><code>Decimal128.prototype.toString()</code></h3>
<p>This method returns a string that is the representation of Decalmai128 value.</p>
<div class="highlight highlight-source-js"><pre><span class="pl-k">let</span> <span class="pl-s1">v</span> <span class="pl-c1">=</span> <span class="pl-c1">0.55</span><span class="pl-s1">m</span><span class="pl-kos">;</span>
<span class="pl-smi">console</span><span class="pl-kos">.</span><span class="pl-en">log</span><span class="pl-kos">(</span><span class="pl-s1">v</span><span class="pl-kos">.</span><span class="pl-en">toString</span><span class="pl-kos">(</span><span class="pl-kos">)</span><span class="pl-kos">)</span><span class="pl-kos">;</span> <span class="pl-c">// prints "0.55"</span></pre></div>
<h3><a id="user-content-decimal128prototypetolocalestringlocale--options" class="anchor" aria-hidden="true" href="#decimal128prototypetolocalestringlocale--options"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><code>Decimal128.prototype.toLocaleString(locale [, options])</code></h3>
<p>This method returns a string that is the locale sensitive representation of Decalmai128 value. We get the same output of applying <code>locale</code> and <code>options</code> to <code>NumberFormat</code> on environments that supports Intl API.</p>

<p>This function returns a string that represents fixed-point notation of Decimal128 value. There is an optional parameter digits that defines the number of digits after decimal point. It follows the same semantis of <code>Number.prototype.toFixed</code>.</p>

<span class="pl-smi">console</span><span class="pl-kos">.</span><span class="pl-en">log</span><span class="pl-kos">(</span><span class="pl-s1">v</span><span class="pl-kos">.</span><span class="pl-en">toFixed</span><span class="pl-kos">(</span><span class="pl-c1">2</span><span class="pl-kos">)</span><span class="pl-kos">)</span><span class="pl-kos">;</span> <span class="pl-c">// prints 0.00</span></pre></div>
<h3><a id="user-content-decimal128prototypetoexponentialfractiondigits" class="anchor" aria-hidden="true" href="#decimal128prototypetoexponentialfractiondigits"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><code>Decimal128.prototype.toExponential([fractionDigits])</code></h3>
